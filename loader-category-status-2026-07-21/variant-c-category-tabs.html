Loader — in-progress · punch-list · close-out gate (Variant C, locked)

Three-part loader UI. Category is metadata + a filter dimension, never a nesting level — hierarchy depth stays at three.

Product goal — eliminate secondary delivery runs. Today loaders don't verify against paper, so missed tags send a truck back. This design's job is to create friction at the departure moment.

Hierarchy: Delivery run = bin X8 → Order = bin.suffix (X8.1, X8.2, X8.3) → status-grouped tags. Category rides along as the "where do I walk" dimension.

How the three parts work together. The in-progress view is where the loader lives; the punch list is the same data pivoted into a walking route; the close-out gate is the only new hard stop.

Rebar shape thumbnails — restored via the existing shapeSVG() function from load_app.html. Reuse, don't fork.

Wrong-bin scan — not silent-fail today. Existing modal (showBinSwitch) already offers Switch/Stay/Stop. This adds Combine as a fourth path and shows both bin summaries side by side.
